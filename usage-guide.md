# Usage Guide

## Overview

This project is a Remote Access Tool (RAT) intended strictly for educational purposes, cybersecurity research, and authorized testing.

It must only be used in controlled environments with proper permission.

---

## ⚠️ Important Warning

- This tool may exhibit behavior similar to malware
- It may be flagged by antivirus software
- It may cause system instability or data loss

DO NOT run this on your main system.

Always use a virtual environment such as VMware or any secure sandbox.

---

## Requirements

- Basic understanding of networking concepts
- Knowledge of operating systems (Windows/Linux)
- Virtualization software (VMware recommended)
- Isolated lab setup

---

## Safe Environment Setup

Before using this tool:

1. Install virtualization software (e.g., VMware)
2. Create a virtual machine (VM)
3. Disable shared folders between host and VM
4. Use a non-personal test OS
5. Avoid using real or sensitive data
6. Keep your VM network isolated or controlled

---

## How to Use (High-Level)

1. Set up your virtual lab environment
2. Review the source code to understand functionality
3. Run the tool only inside the virtual machine
4. Observe behavior for learning and research purposes
5. Stop execution immediately if unexpected behavior occurs

---

## Best Practices

- Always test in isolation
- Never deploy on real-world systems without permission
- Keep your environment disposable (you can reset it anytime)
- Monitor system activity during testing
- Use snapshots to restore VM state

---

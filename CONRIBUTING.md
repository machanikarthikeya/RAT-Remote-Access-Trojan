# Contributing Guidelines

Thank you for your interest in contributing to this project.

This project is intended strictly for educational purposes, cybersecurity research, and authorized testing only. By contributing, you agree to follow all ethical and legal standards.

## Rules for Contribution

1. Ethical Intent Only  
All contributions must align with ethical cybersecurity practices. Any code or suggestion that promotes illegal activity, unauthorized access, or misuse will be rejected.

2. No Malicious Enhancements  
Do not submit features that:
- Enable unauthorized surveillance
- Bypass user consent
- Increase harmful exploitation capability

3. Code Quality  
- Write clean, readable, and well-documented code  
- Follow consistent naming and structure  
- Add comments where necessary  

4. Testing Environment  
All testing must be done in a safe and isolated environment such as :contentReference[oaicite:0]{index=0} or similar virtual machines.

5. Pull Requests  
- Clearly describe what you changed  
- Explain why the change is needed  
- Keep pull requests focused and minimal  

6. Issues  
- Report bugs clearly with steps to reproduce  
- Do not post sensitive or exploitative data  

7. Legal Compliance  
You must comply with all applicable laws. Unauthorized use or testing is strictly prohibited.

## Right to Reject
The project owner reserves the right to reject any contribution that violates these guidelines.

## Responsibility
By contributing, you agree that your work will be used under the project's license and that you take responsibility for your submissions.
